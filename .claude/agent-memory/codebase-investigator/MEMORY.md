## Prebuilt Type System (Investigated 2026-03-05)
See: `prebuilt-type-system.md` for details.

Key gap: `PrebuiltTypeRegistry` in `packages/core/src/prebuilt/types.ts` is a minimum-viable constraint
but the generated `PrebuiltTypes_*` shapes have `typename` on fragments (not in the registry type)
and `void` for fragment input (not assignable to `object` on operations, though `unknown` covers `void`).
The registry type is imported but never used as a `satisfies` constraint in generated files.

## Testing Conventions
- Unit tests colocated with source: `src/**/*.test.ts`
- Type tests: `packages/core/test/types/*.test.ts` (use Bun test runner with type assertions)
- Integration/E2E: `packages/typegen/test/e2e/`
- All test runs via `bun run test`

## Build/Codegen Output Structure
- `_internal.ts` — main schema runtime
- `_internal-injects.ts` — scalar imports
- `_defs/` — chunked type definitions
- `index.ts` — generated prebuilt-typed wrapper (generated by `generateIndexModule`)
- `types.prebuilt.ts` — generated by typegen `emitPrebuiltTypes`, or stub from codegen
